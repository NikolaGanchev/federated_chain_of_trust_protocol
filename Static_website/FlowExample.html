Full Login Flow

Scenario: A user from Bulgaria tries to log into a website. The website only trusts MegaIssuer. The user has a token from their local city clerk.

# Step & Entity Technical Details
1 User requests login
[Browser → RP]		 RP generates a session ID and returns the login page. Page tells the user: 'Prove your nationality to continue.'
2 Client holds local token
[Client (wallet/app)]		 Client checks its token store. It has a JWT issued by blagoevgradissuer.bg. The RP requires a MegaIssuer token. Client needs to exchange upward.
3 Exchange: Local → MegaIssuer
[Client → MegaIssuer] 		POST https://megaissuer.com/exchange_token
MegaIssuer verifies the local token, issues a fresh JWT signed with its own key.
4 Client presents token to RP
[Client → RP]		 POST https://website.com/fctp-callback
The RP sees only a MegaIssuer token. It has no idea the user is from a local city system. It learns: nationality:BG.
5 RP verifies and logs user in
[RP]		 RP calls POST https://megaissuer.com/verify_token. Checks nullifier not already used. User is logged in.
