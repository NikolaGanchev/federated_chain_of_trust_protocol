Federated Chain of Trust

The Federated Chain Of Trust Protocol (FCTP) defines a general-purpose, privacy-preserving framework for identity and attribute verification across the internet. It enables any website to accept verified claims — such as age, nationality, or membership — without implementing custom integrations against government APIs, national ID systems, or third-party identity providers.

Interactive visualisation

This visualisation presents the whole flow of work. The user starts login on the relying party (RP), and the RP asks for proof of some kind of personal information. The local issuer mints a token. Because the RP only trusts the MegaIssuer, the client sends the local token to MegaIssuer to exchange it for a new token. The client sends that MegaIssuer token to the RP. The RP verifies the MegaIssuer token, checks it has not been reused improperly, and then logs the user in.

Privacy by default

Verifying websites learn only the minimum necessary claim. The origin issuer is never revealed.

Minimal integration cost

A website integrates once against the FCTP API and automatically accepts tokens from any issuer.

No mandatory root

There is no central authority. Any entity can be an issuer.

Single-hop exchange

A user exchanges their local token directly at the target issuer.

Format agnostic

Existing identity systems plug in without reimplementing issuance.

Scalable by design

No endpoint serves more than its own immediate relationships.
